C++ Functions

Functions allow to structure programs in segments of code to perform individual tasks.


Some key points about functions in C++.

  • Functions have a return type that specifies the type of data returned by the function.
  • Functions may or may not take arguments as input values.
  • A function declaration tells the compiler about the existence of a function with specific characteristics such as name, parameters (if any), and return value.
  • A function definition provides the actual implementation of a function
  • Functions can either be defined inside the main() function or outside the main() function.
  • There are two types of functions: library functions and user-defined function
  • Recursive function is the one that calls itself within the function body

In C++, a function is a group of statements that is given a name, and which can be called from some point of the program. The most common syntax to define a function is:

type name ( parameter1, parameter2, ...) {{"{"}} statements {{"}"}} Where:
  • Type is the type of the value returned by the function.

  • Name is the identifier by which the function is called.

  • Parameters (as many as needed): Each parameter consists of a type followed by an identifier, with each parameter being separated from the next by a comma. Each parameter looks very much like a regular variable declaration (for example: int x), and in fact acts within the function as a regular variable which is local to the function. The purpose of parameters is to allow passing arguments to the function from the location where it is called from.

  • Statements is the function's body. It is a block of statements surrounded by braces {{"{"}} {{"}"}} that specify what the function actually does.
