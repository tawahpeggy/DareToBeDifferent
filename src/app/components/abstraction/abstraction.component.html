Abstraction

Abstraction is one of the four fundamental concepts in object-oriented programming.
It refers to the practice of hiding the complexity of the underlying implementation and exposing only the essential features of the object.
This allows the user to interact with the object in a simple and intuitive way without needing to know how it works internally.

Here's an example code snippet in C++ that demonstrates abstraction: 
    
        
#include <iostream>
using namespace std;

// Abstract class
class Shape {{"{"}}
   public:
      virtual void draw() = 0; // pure virtual function
{{"}"}};

// Concrete class
class Rectangle : public Shape {{"{"}}
   public:
      void draw() {{"{"}}
         cout << "Drawing a rectangle." << endl;
      {{"}"}}
{{"}"}};
// Concrete class
class Circle : public Shape {{"{"}}
   public:
      void draw() {{"{"}}
         cout << "Drawing a circle." << endl;
      {{"}"}}
{{"}"}};

// Main function
int main() {{"{"}}
   Shape* s1 = new Rectangle(); // Instantiate a rectangle object
   s1->draw(); // Call the draw function on the rectangle object

   Shape* s2 = new Circle(); // Instantiate a circle object
   s2->draw(); // Call the draw function on the circle object

   return 0;
{{"}"}}

In this example, we have an abstract class called Shape that contains a pure virtual function draw().

This means that any class that inherits from Shape must implement the draw() function.

We then have two concrete classes, Rectangle and Circle , that inherit from Shape and provide their own implementation of the draw() function.

Finally, in the main function, we create objects of type Rectangle and Circle and call their draw() functions using a pointer of type Shape*.

This allows us to interact with the objects in a simple and intuitive way without needing to know the underlying implementation details.
